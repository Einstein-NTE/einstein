1.1. Installation of EINSTEIN Version 1.0 for Newcomers

If You already have successfully installed some of the preliminary versions of EINSTEIN V1.00a or V1.00b please jump to section 1.2.

1.1.1. Basic set-up previous to the installation of EINSTEIN

Einstein uses the following software packages that have to be installed PREVIOUSLY to the installation of EINSTEIN itself :
- MySQL (Version 5.1 or higher)
- Python (Version 2.5 or higher)
- Several Python complements:
    a) wxPython
    b) matplotlib
    c) numpy
    d) MySQL-Python

HOW TO INSTALL THESE PACKAGES ?

Step 1: Create your einstein installation folder, e.g. C:\MyEINSTEIN.

Step 2: Copy the following files into C:\MyEINSTEIN and then unzip:
- EINSTEIN_1.0.zip
- MySQLandPython.zip

Please CHECK: After this you should have (at least) the following files and folders in your "MyEINSTEIN" directory:
- EINSTEIN.bat
- install_mysql.bat
- install_python.bat
- install_einstein.bat
- check_Installation.bat
- folder "einstein"
- folder "installer"

Step 3: Set-up of MySQL (skip if you have already MySQL 5.1 or higher running, skip this section)

TAKE CARE: Uninstall or deactivate any older Version of MySQL BEFORE installing the new Version !!!
Parallel installation of two different MySQL Versions may lead to problems.
IMPORTANT: if You already had an old version of MySQL installed, after uninstalling You should also delete manually the remaining files named "mysql" on your computer.
Those are usually the following (in WINDOWS installations):
C:\Program Files\mysql
C:\Documents and Settings\All Users\Application Data\mysql
C:\Doucments and Settings\<Individual User's Name>\Application Data\mysql

There are two possible ways to doing MySQL installation. We recomment use of the installer (option (a) ), but in case of any problem you can try it with option (b).

a) Run the file "install_mysql.bat" and follow the instructions given on the screen

VERY IMPORTANT:
Choose all the default options offered by the MySQL installer except the following (in the order that they appear):
- select the option Port Number 3306: "add firewall exception for this port"
- select the option: include bin-directory in Windows Path
- UNselect the option: modify security settings (so that You can install MySQL without password)

b) go to: http://dev.mysql.com/downloads/mysql/5.1.html and download the Windows ZIP/Setup.EXE (x86)
(or one of the other versions, depending on your computer and operating system)

Take care of activating / deactivating the MySQL configuration settings as specified under (a)

CHECK OF MySQL INSTALLATION (On Windows)

To check the correct installation of MySQL, You can open the Control Panel, and look for an icon called Administrative Tools.
Open that, and there is another icon called Services. Open that too.
You will see a list of programs running in your machine. The list is ordered alphabetically.
Look for MySQL. It should be marked as Running (or Initiated or something like that) and in the next column, it should be marked as Automatic.
These settings ensure that MySQL is started each time you start your computer.
If it is there, but not running, click on the line with the Right button and choose Start. This will start MySQL, but only this once.
To make it start always, choose Properties. There you can select Automatic.
If MySQL refuses to run, please first deinstall and then reinstall it.

Step 4: Set-up of  Python and complements

There are two possible ways to doing Python installation. We recomment use of the installer (option (a) ), but in case of any problem you can try it with option (b).

a) Run the file "install_python.bat" and follow the instructions given on the screen

b) Go to www.python.org/download and do it yourself.

The installers for the necessary complements you find at:

wxPython: http://www.wxpython.org/
numpy: http://numpy.scipy.org/
matplotlib: http://matplotlib.sourceforge.net/
MySQL-Python: http://sourceforge.net/project/showfiles.php?group_id=22307

Install also these additional packages on your computer.

Step 5: Check Your Firewall

Do you have a firewall on your machine?
If so, you should free port 3306, which is used by MySQL

==> HOW CAN I DO THAT?

In general firewalls have an icon on the bottom right part of your desktop. It says 'security' or 'firewall' or something like that.
When you open it, there usually is some button called 'advanced settings' or something.
A precise answer to your question would depend on what firewall do you have installed.

-------------------------------------------------------------------

If there are problems with MySQL but it is not a problem with the firewall, please try this:
Open a command window and write the command:

NET START mysql

This should show a more descriptive error diagnostic.

Step 6: Check Your Anti-Virus configuration

Some Anti-Virus software, like e.g. McAfee may cause problems in combination with the MySQL database server.
MySQL creates some temporary files in the folder C:\Windows\Temp, and some antivirus software does not allow MySQL to delete these temporary files.
This can be solved excluding the folder C:\Windows\Temp from real-time scanning in the Antivirus (in McAfee: option All Processes / detection / exclusions).

1.1.2. Installation of EINSTEIN

Step 1: Installation of the EINSTEIN database

There are two possible ways to doing the EINSTEIN database installation. We recomment use of the installer (option (a) ), but in case of any problem you can try it with option (b).

a) Run the file "install_einsteinDB.bat" and follow the instructions given on the screen.

b) Run EINSTEIN (see below) and use the Menu-Option:

Database -> Database Administration -> Restore Database

You should select the file "einstein_V1.0.sql" in the folder <MyEINSTEIN>\einstein\sql

Step 2: Now have a try -> Run EINSTEIN

For any problems check the FAQ Section of this manual.

Step 3: Configuration of MySQL password:
If you configured MySQL with a password, you have to specify this password in the Einstein configuration:

There are two possibilities
a) edit manually the file einstein\GUI\einstein.ini and put the password there. You will see that there is no password, originally.

b) Use the option  "Database -> Database Administration" in EINSTEIN, and enter your password there.
NOTE: you have to specify the folder where the executables of MySQL are located
(typically something like C:\Program Files\MySQL\MySQL Server 5.1\bin)
  

1.2. Update from previous Versions

Step 1:  You can use your existing EINSTEIN installation folder, e.g. C:\<MyEINSTEIN> , or you can create a new one, e.g. C:\<MyEINSTEIN V1.0>

Step 2: Copy the file "EINSTEIN_V1.0.zip" into C:\<MyEINSTEIN> and then unzip.
(if you use the folder of an existing installation, this may overwrite some of the old files. When asked, confirm overwriting files with OK).

Please CHECK: After this you should have (at least) the following files and folders in your "MyEINSTEIN" directory:
- EINSTEIN.bat
- install_einsteinDB.bat
- folder "einstein"

Step 3: Install the update of the EINSTEIN database (-> section 1.1.2).

TAKE CARE: If You have a previous installation of EINSTEIN running on your computer and do not want to lose the data in your database, you should NOT carry out the database installation, as this will overwrite existing data !!!

For updating your database STRUCTURE to Version 1.0 without losing data you can do it by several ways:

a) run the file "update_einsteinDB.bat"

or

b) by hand:

Open the MS-DOS command shell on your computer.

enter in the folder
<MyEinstein>\einstein\sql

Type the following

mysql ----host=localhost --user=<username> --password=<password> < update_einsteinDB_V1.00b_to_V1.0.sql
(if you have a password specified on mysql)

or
mysql ----host=localhost --user=<username>  < update_einsteinDB_V1.00b_to_V1.0.sql
(without password)

where
<username> is your username on the mysql server (by default: root)
<password> is your password on mysql

if you get the error message "mysql not found", then you have first to add MySQL to the PATH on your computer:

set PATH = PATH; "C:\<path to your mysql installation>\MySQL\MySQL Server5.1\bin"


Step 4: Now have a try -> Run EINSTEIN

For any problems check the FAQ Section of this manual.
