Chapter 2.12: Running your first projects with EINSTEIN

 
Within the default database in EINSTEIN you can find some example projects:

§
§
1. EINSTEIN Container Washing Ltd.

Example of an industry that has only one thermal process: a washing process consuming hot water at 80 ºC

2. EINSTEIN Milk Products Inc.

Example of an industry with 3 different thermal processes

2.12.1. Basic rules

a) NEVER work directly within the example projects. Make your first steps on a COPY !!!

=> enter in the menu Edit industry data, select one of the example projects (e.g. EINSTEIN Container Washing Ltd.), and press the button "copy project".
Give a new name to the copy, and press OK. Then the new project will appear automatically as selected on the status bar of the tool

b) from time to time make a back-up of your data (Menu "DataBase -> Database Administration -> Backup Database").

2.12.2. Guided tour with the EINSTEIN auto-pilot

Make your copy of  the example project "EINSTEIN Container Washing Ltd." or open the project "TRY IT YOURSELF: EINSTEIN Container Washing".
Press the button "Run EINSTEIN Audit procedure"

The tool automatically will guide you throught the audit steps, from consistency checking to proposal generation, and will finally show you the comparative statistics of the different alternatives.
Just press OK every time you are asked for.

On this guided tour You will see some of the most important screens of the EINSTEIN tool, such as the design assistants of the different sub-systems.

Once succeeded, do the same with the project EINSTEIN Milk Products Inc.

2.12.3. Make Your own projects

Start using (A COPY !!!) of one of the example projects. Once you have got some experience, You can start building a new project from scratch (Option Edit Industry Data -> New Project)

Then modify the input data (e.g. increase the heat demand of some process or add a new one, etc.) in the submenus of "Edit Industry Data".

a) consistency check module

The first barrier You have to overcome is to make Your input data consistent and to learn to work with the consistency check module.
=> Menu "Consistency Check", press button "Basic check"

As a beginner, do only change ONE parameter a time within the questionnaire, and then make the thing consistent again, until you get a feeling on how that works ...

If You entered too few data and there are parameters missing, you can try with the "estimate data" option. If this does not help, there's no other way than going back to the questionnaire and feeding EINSTEIN with more complete information.
