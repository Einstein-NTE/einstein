1. Problems with the set-up of EINSTEIN

1.1. Problems with the set-up of MySQL

If there is any problem with MySQL installation, check the following:

Run the file "checkInstallation.bat" in your <MyEINSTEIN> folder, and take note of the possible error messages.

If there's a problem, try to install MySQL again manually from the MySQL web page.

1.2. Problems with the installation of Python

If there is any problem with Python installation, check the following:

Run the file "checkInstallation.bat" in your <MyEINSTEIN> folder, and take note of the possible error messages.

If there's a problem, try to install Python and complements again manually from the Python web page.


1.3. Problems loading the EINSTEIN database

Q: Data base connection error

The installation of EINSTEIN was carried out with success. EINSTEIN starts, but a "connection error" with the einstein database occurs

A: First check if the einstein database is installed well in MySQL. If everything is ok, check the configuration of the parameters in the menu databases->database administration.
The default configuration is:

host: localhost
database name: einstein
user name: root
password: none (leave blank space)

Use "test connection" in order to see whether EINSTEIN can connect with the database. If there's any problem, check the configuration parameters.

2. EINSTEIN has been installed with success, but doesn't start-up

2.1. Problems with the connection to the MySQL database
2.1.1. Connection error: (1045, "Access denied for user 'root'@'localhost' (using password: NO)")
This error usually occurs", when some wrong option was chosen when installing MySQL.

The solution: Please uninstall and reinstall MySQL again, following exactly what is written in Section 1.1.1., Step 3 of the User Manual,
especially:
- make sure that when uninstalling you REMOVE MANUALLY all the remaining mysql folders on your computer
- when installing again, make sure to UNCHECK the option "modify security settings"

Note also, that after having re-installed MySQL, you also have to load the einstein data base again (with install_einsteinDB.bat)

Where do You get help ? Where to report bugs ?


For getting help with EINSTEIN you can also subscribe and send a mail to the mailing list of EINSTEIN users: https://lists.sourceforge.net/lists/listinfo/einstein-users

For any reporting of possible bugs and problems enter a bug report at http://sourceforge.net/tracker2/?func=add&group_id=213015&atid=1024025
or send a mail to the EINSTEIN team.

Please include the files einsteinMSG.txt, einsteinLOG.txt and einsteinTRACK.txt, and a short description of
a ) which steps you carried out BEFORE the problem arose
b) when did the problem occur
c) what was the problem
